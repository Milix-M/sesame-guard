# SESAME Guard - 機能仕様書

## 概要

SESAMEスマートロックの解錠イベントを監視し、リスクの高い解錠を検知してLINEに通知するセキュリティ監視ツール。

## 前提条件

- SESAMEスマートロック（CANDY HOUSE製）
- Wi-Fi Access Point（Hub3）が設定済み
- SESAME API v3が有効（Candy House Dashboard → Integration ON）
- OCI Compute Instance上で稼働

---

## 1. ポーリング機能

### 1.1 状態取得
- SESAME Web API v3の `GET /sesame/{device_id}` を定期呼び出し
- デフォルト間隔: 60秒（環境変数 `POLL_INTERVAL_SEC` で変更可）
- 取得項目: `locked`（bool）, `battery`（int）, `responsive`（bool）

### 1.2 デバイス自動検出
- `SESAME_DEVICE_ID` 未設定時、`GET /sesames` で一覧取得し、最初のデバイスを使用

### 1.3 状態変化検知
- 前回の `locked` 状態と比較
- **locked → unlocked** の変化のみを「解錠イベント」として扱う
- unlocked → locked の変化は記録するが通知はしない

---

## 2. リスク判定ルール

### 2.1 深夜帯の解錠（平日/休日 differentiated）

| 項目 | 値 |
|------|-----|
| 対象時間帯（平日） | 2:00 〜 5:00（JST） |
| 対象時間帯（休日） | 3:00 〜 5:00（JST） |
| 環境変数 | `RISK_NIGHT_START`, `RISK_NIGHT_END` |
| 判定条件 | 解錠イベントの発生時刻が対象時間帯に含まれる |
| 検知理由文 | 「深夜帯の解錠（平日/休日 {H}:00台）」|

休日は平日より1時間遅い開始（3:00〜）を適用。週末の帰宅が遅くなるケースを考慮。

> **✅ JST対応済**: 時間判定はJST（UTC+9）基準で動作します。

### 2.2 短時間の連続解錠

| 項目 | 値 |
|------|-----|
| 監視期間 | 300秒（5分） |
| 閾値 | 3回以上の解錠 |
| 環境変数 | `RISK_RAPID_THRESHOLD_SEC`, `RISK_RAPID_COUNT` |
| 判定条件 | 監視期間内の解錠イベント数が閾値以上 |
| 検知理由文 | 「短時間に{N}回の解錠（{SEC}秒以内）」 |

### 2.3 統計的異常検知（パターン学習）

| 項目 | 値 |
|------|-----|
| 学習期間 | 過去30日間 |
| 最低データ数 | 10件（未満の場合はスキップ） |
| 閾値 | 2σ（標準偏差の2倍以上の乖離） |
| 環境変数 | `RISK_ANOMALY_SIGMA` |

**判定方法：**

1. **ゼロヒット検知**: 過去30日間でその時間帯（1時間単位）に解錠実績が0回 → 即座に異常
   - 検知理由文: 「過去30日で解錠実績のない時間帯（{H}:00台）」
2. **Z-score検知**: 解錠時間の分布から平均・標準偏差を計算し、現在時刻が2σ以上外れている場合に異常（時間は循環距離で計算）
   - 検知理由文: 「通常と異なる時間帯の解錠（{H}:00台、通常は{MEAN}時台付近）」

**特長:**
- 機械学習のライブラリ不要（純粋な統計計算）
- データが蓄積されるほど精度が上がる
- 利用者の生活パターンに自動適応

### 2.4 長時間未施錠検知

| 項目 | 値 |
|------|-----|
| 閾値 | 120分（2時間） |
| 環境変数 | `RISK_UNLOCK_TIMEOUT_MIN` |
| 判定条件 | 解錠状態が閾値以上継続 |
| 検知理由文 | 「長時間未施錠（{MIN}分経過）」 |
| 通知頻度 | タイムアウト1回のみ（施錠でリセット） |

締め忘れや、解錠されたまま放置されている状況を検知。

### 2.5 判定の優先度

全ルールを並列実行。複数のルールにマッチした場合、全ての理由を通知に含む。

### 2.6 通知クールダウン

| 項目 | 値 |
|------|-----|
| クールダウン | 30分 |
| 環境変数 | `RISK_COOLDOWN_MIN` |
| 動作 | 最後の通知からクールダウン時間内は通知を抑制 |

深夜に同じ理由で大量の通知が来るのを防止。イベント自体はDBに記録される。

## 3. 通知機能

### 3.1 通知先
- LINE Messaging API（プッシュメッセージ）

### 3.2 通知フォーマット

**リスク検知時:**
```
🚨 SESAME セキュリティ警告
解錠検知: 2026-03-28 02:15:00 JST
判定理由:
  ‣ 深夜帯の解錠（平日 2:00台）
  ‣ 過去30日で解錠実績のない時間帯（2:00台）
🔋 バッテリー: 85%
```

**長時間未施錠時:**
```
⚠️ SESAME 長時間未施錠警告
解錠から120分が経過しています
施錠忘れの可能性があります
🔋 バッテリー: 85%
```

### 3.3 通知タイミング
- リスク判定が `True` の場合（クールダウン対象外なら即時）
- 未施錠タイムアウト検知時（1回のみ）
- 通常の解錠（リスクなし）はDB記録のみ

## 4. データ管理

### 4.1 lock_state テーブル

| カラム | 型 | 説明 |
|--------|-----|------|
| id | INTEGER | 常に1（単一行） |
| locked | BOOLEAN | 現在の施錠状態 |
| updated_at | TEXT | 最終更新日時（ISO 8601 / JST） |

### 4.2 events テーブル

| カラム | 型 | 説明 |
|--------|-----|------|
| id | INTEGER | 自動採番 |
| event_type | TEXT | `unlock` / `status_change` |
| locked | BOOLEAN | イベント時の状態 |
| battery | INTEGER | バッテリー残量 |
| is_risky | BOOLEAN | リスク判定結果 |
| reason | TEXT | リスク理由（nullable） |
| created_at | TEXT | イベント日時（ISO 8601 / JST） |
| created_at_jst | TEXT | JST日時（インデックス用） |

### 4.3 alert_log テーブル

| カラム | 型 | 説明 |
|--------|-----|------|
| id | INTEGER | 自動採番 |
| reasons | TEXT | 通知理由（改行区切り） |
| created_at | TEXT | 通知日時（ISO 8601 / JST） |

クールダウン判定に使用。

### 4.4 データ保持
- 現状、自動削除なし（運用に応じて追加予定）

## 5. 環境変数一覧

| 変数名 | 必須 | デフォルト | 説明 |
|--------|------|-----------|------|
| `SESAME_API_TOKEN` | ✅ | - | SESAME API認証トークン |
| `SESAME_DEVICE_ID` | ❌ | 自動検出 | SESAMEデバイスID |
| `LINE_CHANNEL_ACCESS_TOKEN` | ✅ | - | LINE Messaging API トークン |
| `LINE_USER_ID` | ✅ | - | 通知先LINEユーザーID |
| `RISK_NIGHT_START` | ❌ | `2` | 深夜判定開始時（時、JST） |
| `RISK_NIGHT_END` | ❌ | `5` | 深夜判定終了時（時、JST） |
| `RISK_RAPID_THRESHOLD_SEC` | ❌ | `300` | 連続解錠の監視期間（秒） |
| `RISK_RAPID_COUNT` | ❌ | `3` | 連続解錠の閾値（回） |
| `RISK_UNLOCK_TIMEOUT_MIN` | ❌ | `120` | 未施錠警告の閾値（分） |
| `RISK_COOLDOWN_MIN` | ❌ | `30` | 通知クールダウン（分） |
| `RISK_ANOMALY_SIGMA` | ❌ | `2.0` | 異常検知のσ閾値 |
| `POLL_INTERVAL_SEC` | ❌ | `60` | ポーリング間隔（秒） |
| `DB_PATH` | ❌ | `sesame_guard.db` | SQLiteファイルパス |
